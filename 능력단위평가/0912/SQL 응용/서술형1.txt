문제 : SQL 프로시저와 함수의 차이점 설명

답안: 

함수 : 하나의 특별한 목적의 작업을 수행하기 위해 독립적으로 설계된 코드의 집합

함수는 주로 단일 값을 계산하거나 반환하는 데 사용됩니다. 클라이언트(화면)에서 값을 건네 받고 서버에서 필요한 값을 가져와서 클라이언트에서 작업을 하고 반환합니다. 그래서 프로시저보다는 속도가 느립니다.


프로시저 : 일련의 쿼리를 마치 하나의 함수처럼 실행하기 위한 쿼리의 집합이며, 일련의 작업을 정리한 절차

프로시저는 서버에서 실행되어 복잡한 작업을 빠르게 수행하고 , 반환값이 필요하지 않거나 여러 값을 처리해야 할 때 적합합니다. 프로시저는 트랜잭션 처리가 가능하고, IN, OUT 매개변수를 통해 다양한 데이터를 주고받을 수 있습니다.
